# [Feature] 增强内容朗读：支持后台媒体控制、有限连续播放队列和自定义朗读模板

## 你想解决什么问题？

目前的朗读功能更接近“朗读当前这一条内容”：

- 只支持回答和文章，不支持想法。
- 只能开始或停止，不能真正暂停后继续。
- 离开页面、切到后台或锁屏后缺少稳定的播放体验。
- 无法通过系统通知栏、锁屏界面、耳机或蓝牙设备控制上一条、暂停/继续和下一条。
- 每次只能手动打开并朗读一条内容，播放完成后不会根据当前页面上下文继续朗读。
- 朗读内容固定，不能决定是否朗读作者、发布时间、点赞数、评论等信息，也不能调整顺序。

在通勤、做家务、运动、视力不便或不方便持续查看屏幕时，用户希望像使用播客或音乐播放器一样连续收听知乎内容，而不需要每读完一条就重新操作手机。

## 你希望怎么改进？

希望将现有朗读功能升级为一个统一的“连续朗读播放器”。一次朗读会话由当前内容、来源页面、有限播放队列和用户配置的朗读模板组成。

### 1. 后台播放与 Android 系统媒体控制

- 朗读开始后，即使应用进入后台、切换页面或设备锁屏，也应继续播放。
- 接入 Android 系统媒体会话，支持从以下位置控制播放：
  - 系统媒体通知
  - 锁屏播放器
  - 耳机或蓝牙设备
  - Android 系统媒体控制面板
- 至少支持：
  - 上一条
  - 暂停/继续
  - 下一条
  - 停止播放
- 暂停后继续时，应从当前句子或当前分段附近恢复，而不是重新朗读整条内容。
- 系统媒体信息中应显示当前内容的标题、作者和内容类型。
- 应正确处理系统音频焦点，例如来电或其他应用开始播放音频时暂停或降低音量。

具体实现可以由维护者决定，例如使用 MediaSession/Media3 与前台媒体服务。

### 2. 支持更多内容类型

统一支持朗读：

- 回答
- 专栏文章
- 想法

不同内容类型缺少某个字段时，应自动跳过该字段。例如想法没有标题时，不应朗读空标题或占位文本。

### 3. 自定义朗读内容及顺序

用户可以选择需要朗读的字段，并调整字段顺序。可配置字段包括：

- 内容类型，例如“这是一条回答”
- 标题或问题标题
- 作者名称
- 正文
- 发布时间
- 点赞数
- 评论区内容

评论区应支持：

- 完全不朗读评论
- 自定义朗读前 N 条评论
- 按当前评论排序读取
- 每条评论至少朗读评论作者和评论正文
- 仅在启用评论朗读时请求评论数据

字段可以通过拖动等方式排序。例如用户可以配置为：

> 内容类型 → 标题 → 作者 → 发布时间 → 正文 → 点赞数 → 前 3 条评论

### 4. 根据当前页面生成有限播放队列

用户开始朗读时，以当前正在查看或选中的内容作为播放队列第一项，并根据来源页面继续添加后续内容。

例如：

- 在问题页面开始朗读回答：
  - 当前回答作为第一项
  - 后续项目为该问题当前排序下的后续回答
- 在推荐页面开始朗读：
  - 当前推荐内容作为第一项
  - 后续项目为推荐列表中接下来的内容
- 其他具有明确顺序的页面，例如收藏夹、搜索结果、用户内容列表、热榜、历史记录等，也应能够按照当前列表顺序生成队列。

队列来源不应写死为问题页和推荐页，而应设计成可由不同列表页面提供后续内容。

用户可以设置每次最多加载多少条，例如 5、10、20 条或自定义数量。这个数量包含当前内容。

队列必须是有限的：

- 可以按需加载，但不能超过用户设置的最大数量。
- 播放到队尾后自动结束，不应无限请求下一页并持续播放。
- 如果当前来源不足指定数量，则使用实际可获得的内容。
- 单条内容加载失败时，应提示或跳过，不应导致整个播放队列中断。

### 5. 自定义内容之间的过渡文本

用户可以配置每两条内容之间朗读的过渡文本，也可以留空以关闭过渡文本。

例如：

> 本条内容朗读完毕，接下来为：{title}，作者：{author}。

建议支持以下占位符：

- `{index}`：下一条在队列中的序号
- `{total}`：队列总数
- `{contentType}`：回答、文章或想法
- `{title}`：下一条标题
- `{author}`：下一条作者

第一条之前和最后一条之后不需要朗读条目间过渡文本。

### 6. 应用内播放工具栏与队列界面

只要朗读会话仍然存在，包括正在播放和暂停状态，页面底部显示一整排播放器工具栏。

工具栏至少包含：

- 上一条
- 暂停/继续
- 下一条
- 朗读设置
- 播放列表

播放列表可以展开为底部弹层或独立页面，并满足：

- 显示本次有限队列中的所有项目。
- 显示标题、作者和内容类型。
- 高亮当前播放项目。
- 显示当前位置，例如“3 / 10”。
- 点击任意项目后，立即切换并播放该项目。
- 在应用内点击项目时，同时跳转到相应的回答、文章或想法页面。
- 应用内工具栏、播放列表和系统媒体控制中的状态保持同步。

播放到首项时“上一条”不可用；播放到末项时“下一条”不可用。

## 验收标准

- [ ] 回答、文章和想法都可以开始朗读。
- [ ] 应用进入后台或设备锁屏后仍能继续播放。
- [ ] 通知栏、锁屏和耳机可以控制上一条、暂停/继续和下一条。
- [ ] 暂停后继续不会从整条内容开头重新朗读。
- [ ] 用户可以启用、禁用并调整朗读字段的顺序。
- [ ] 评论朗读数量遵循用户配置，关闭后不会额外加载评论。
- [ ] 从当前内容及其来源页面生成有限队列。
- [ ] 队列长度不会超过用户设置，也不会无限加载。
- [ ] 问题页、推荐页及其他有序内容列表可以通过统一方式提供后续内容。
- [ ] 自定义过渡文本会在相邻项目之间朗读。
- [ ] 播放工具栏在播放和暂停期间持续显示。
- [ ] 展开队列后可以查看、选择并跳转到任意项目。
- [ ] 单条内容加载失败不会导致整个播放会话崩溃。

## 使用场景

- 通勤、步行、运动或做家务时，连续收听推荐内容。
- 锁屏后通过耳机控制播放，不需要频繁取出手机。
- 在问题页面连续听完多个回答。
- 用户只关心正文时，可以关闭作者、时间、统计和评论。
- 希望了解讨论情况时，可以在正文后追加前几条评论。
- 视力不便或长时间阅读容易疲劳的用户，可以将知乎++作为连续内容播放器使用。

## 官方操作路径（如果官方有此功能）

本建议不要求完全复刻知乎官方的某一个入口，主要参考音乐、播客和有声阅读应用的通用媒体播放交互。

## 你尝试过的替代方案

目前可以逐条进入回答或文章并手动开始朗读，也可以使用 Android 的屏幕朗读或无障碍服务。

这些方案无法根据知乎++当前页面生成连续队列，也无法自定义朗读字段、评论数量和内容之间的过渡文本。

## 知乎++ 版本

`0.24.4（222）`

如实际安装版本不同，提交前替换成设置页面显示的版本。

## 其他补充

虽然需求包含多个组成部分，但它们都服务于同一个“连续朗读播放器”能力，可以分阶段实现：

1. 后台服务、暂停恢复与系统媒体控制。
2. 有限播放队列及不同页面的队列来源。
3. 想法支持与自定义朗读模板。
4. 应用内播放器工具栏、队列界面和评论朗读。

首版不要求无限播放、离线生成音频文件或应用进程被系统完全终止后的队列恢复。
